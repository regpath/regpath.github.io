지식 확장: 검색 증강 생성 (RAG)

왜 RAG가 필요한가?

LLM은 훈련 데이터 시점 이후의 정보를 모르고, 때로는 사실이 아닌 내용을 생성합니다 (환각). RAG는 외부의 최신/신뢰성 있는 데이터를 참조하여 이 문제를 해결합니다.

LLM 활용 사례의 대다수는 외부 데이터 연동이 필수적입니다.

기본 RAG 파이프라인 6단계

1. 로딩

문서(PDF, TXT)를 시스템으로 가져옵니다.

→
2. 분할

긴 문서를 검색에 용이한 작은 '청크'로 나눕니다.

↓
4. 저장

임베딩 벡터를 '벡터 저장소'에 색인합니다.

←
3. 임베딩

텍스트 청크를 의미를 담은 숫자 벡터로 변환합니다.

↓
5. 검색

질문과 가장 유사한 청크를 벡터 저장소에서 찾습니다.

→
6. 생성

검색된 정보와 질문을 함께 LLM에 전달해 답변을 만듭니다.

RAG 레벨 업: 고급 아키텍처

기본 RAG의 검색 정확도와 컨텍스트 이해도를 높이기 위한 핵심 전략입니다.

상위 문서 리트리버

"작은 것으로 검색하고, 큰 것으로 이해한다"

작은 텍스트 조각(하위 청크)으로 정확하게 검색한 뒤, LLM에게는 해당 조각이 포함된 더 큰 원본 문단(상위 청크)을 제공하여 풍부한 맥락 속에서 답변을 생성하게 하는 기법입니다.

재순위화 (Re-ranking)

"가장 관련성 높은 정보만 걸러낸다"

초기 검색에서 찾은 여러 후보 문서들을 더 정교한 '재순위화 모델'로 다시 평가합니다. 이를 통해 노이즈를 제거하고 질문과 가장 밀접한 문서만 선별하여 LLM에 전달합니다.

궁극의 맞춤화: 미세조정 (Fine-Tuning)

언제 미세조정을 사용할까?

지식(Knowledge) 주입이 필요할 때 → 🧠 RAG

모델이 특정 도메인의 최신 정보를 '알게' 해야 할 경우.

기술/스타일(Skill/Style) 학습이 필요할 때 → 🛠️ 미세조정

모델의 '행동'을 바꿔 특정 말투나 출력 형식을 따르게 할 경우.

미세조정의 혁신: QLoRA

QLoRA는 거대 모델의 가중치를 압축(양자화)하여, 소비자용 GPU에서도 미세조정이 가능하게 만든 혁신적인 기술입니다. VRAM 요구사항이 극적으로 감소합니다.
